1. **MEDIUM — Pre-push gate hides vite build errors**
   - Description: `(cd apps/desktop && npx vite build > /dev/null 2>&1)` redirects both stdout *and* stderr to `/dev/null`. On failure the user only sees `✗ pre-push gate failed at: vite build` with zero context — no missing module, no TS slop, no syntax error. This is exactly the case where the gate needs to *help* the developer fix the issue, not just block them. 2. **MEDIUM — Pre-commit doesn't `cargo check` the CLI, so CLI breakage is only caught at push time**
   - Description: Pre-commit runs `cargo check` on `apps/desktop/src-tauri` only. Pre-push runs `cargo build --release` on `apps/cli`. A commit that breaks the CLI compiles fine locally, then the developer hits push and waits 30s–2 min for a release build to tell them. The gate's stated purpose is "catch type errors before they get committed" — half the Rust surface isn't being typechecked. Adding `(cd apps/cli && cargo check --quiet)` to pre-commit is essentially free (incremental cache hits in single digits of seconds) and shifts the feedback left.
   - Location: `.githooks/pre-commit:18-19`
   - Fix: Add a `cargo check (apps/cli)` step between the existing cargo check and vitest steps.
   - Verified-via: `read_file .githooks/pre-commit`, `read_file .githooks/pre-push`

3. **MEDIUM — `fire_mode` validated on write but not on read; unknown values fall through to "fire always"**
   - Description: `save_agent_hook` enforces `fire_mode ∈ {always, keyword, llm-decides}` (good). `list_agent_hooks` reads whatever string is in the DB without validation, and `should_fire_hook` in `mod.rs` has a final `else { true }` branch (line 9421) — so a DB row with `fire_mode = "garbage"` will always fire and ignore its `whenKeywords` / `whenDescription` rules. That's the *unsafe* failure mode (over-firing leaks data and burns tokens, vs. the safer skip). Possible sources of garbage: hand-edited DB, future schema additions written by a newer build, a migration from a still-unsupported mode.
   - Location: `apps/desktop/src-tauri/src/commands/agent_hooks_evals.rs:95` (read with no validation) + `apps/desktop/src-tauri/src/commands/mod.rs:9421` (default to true)

   - Verified-via: `read_file apps/desktop/src-tauri/src/commands/agent_hooks_evals.rs:85-100`, `read_file apps/desktop/src-tauri/src/commands/mod.rs:9366-9424`

   - Description: The smoke loop calls `$ATO $cmd > /dev/null 2>&1` with no timeout. `runtimes list-remote` and `mesh discovered` plausibly touch the network.

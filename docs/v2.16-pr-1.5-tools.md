# v2.16 PR-1.5 — API-provider write/edit tool surface

> **Status:** war-room locked 2026-06-12 (war_room_id `74B2FBE8-4F45-4E09-9F6B-E809BD458FD7`, codex 2x + gemini 1x). PR-1.5 implementation in flight.

## Why this exists

Today the API-provider tool loop (anthropic/openai/google/minimax via `apps/cli/src/api_dispatch_tools.rs::dispatch_with_tools`) supports only read tools — `read_file`, `grep`, `git_log`. CLI runtimes (claude/codex/gemini CLI binaries) can edit/write code via their own native tool loops; API providers can only review.

Per the [v2.16 Missions design](v2.16-missions.md) — driver clarified that Missions coordinate multiple coding agents working in different worktrees. **Without write/edit tools on API providers, Missions are limited to a 3-runtime team** (claude/codex/gemini CLI). The Fable-cost-shift value (cheapest-model-per-step across ALL providers) is gutted.

PR-1.5 is a **PR-4 blocker** — the coordinator tick must be able to dispatch ANY runtime as a worker, not just CLI runtimes.

## What ships

Six new tools added to `packages/ato-review-tools/src/lib.rs::registry()`:

| Tool | Purpose | Schema |
|---|---|---|
| `edit_file` | Surgical text replacement | `{path, old_string, new_string, replace_all?, expected_replacements?}` |
| `write_file` | Create or overwrite a file | `{path, content, overwrite_existing?}` |
| `list_dir` | Inspect directory structure | `{path?, include_hidden?}` |
| `git_status` | Current working-tree status | `{}` |
| `git_diff` | Diff vs HEAD or a ref | `{ref?, path?, staged?}` |
| `bash` | Scoped shell exec | `{command, cwd?, timeout_secs?}` |

All tools sandboxed via the existing `sandbox_path()` primitive (rejects abs paths / `..` / repo escape). `bash` adds a first-token allowlist + denylist + **parsed-argv enforcement** (Q7a, see below).

## War-room verdicts (full)

### Q1 — bash allowlist initial set

[REWORK with note] — adopt BOTH codex and gemini additions.

**Allowlist:**
```
# Build / package tools
cargo, npm, pnpm, yarn, npx, node, bun, deno, tsc, eslint, prettier
# Test runners
pytest, jest, vitest
# Languages / package managers
python, python3, ruby, bundle, go, ruff, uv
# Build systems
make, mvn, gradle
# File inspection
ls, cat, find
# File creation (sandbox_path scoped)
mkdir, cp
# ato (read-only subcommands only)
ato
# git (read-only subcommands only)
git
```

**Hard denylist** (refused even if first-token is allowed):
- Any `sh -c` / `bash -c` / `zsh -c` shape — Q7a enforcement
- `rm`, `mv` (out of repo), `curl`, `wget`, `nc`, `ssh`, `scp`
- `sudo`, `chmod`, `chown`
- `git push`, `git reset --hard`, `git config`, `git filter-branch`, `git checkout` (mutating refs)
- `ato dispatch`, `ato war-rooms`, `ato missions tick/dispatch/merge`, `ato replay start`
- Network install: `npm install`, `pnpm install`, `yarn add`, `pip install`, `uv pip install`, `cargo install`, `go install`, `gem install` (mission opt-in via `allow_network_install: true` queued for PR-2)
- Anything referencing `~/.ssh`, `~/.aws`, `/etc/`

### Q2 — `edit_file` uniqueness

**[B with gemini override]** — Claude Code shape: `old_string` must be unique by default; `replace_all=true` bypasses with `expected_replacements: N` requirement for safety.

Codex argued strict-only; rejected because rename refactors are common and forcing read+write_file for them is friction without security gain. Codex's "weak visibility" concern handled by `expected_replacements` — mismatch fails the call.

### Q3 — `write_file` overwrite

**[B]** — explicit `overwrite_existing: true` required to overwrite an existing file. Default is create-only. Matches Mission live-proof pattern.

### Q4 — `bash` timeout

**[B]** — 120s default, 600s ceiling. Caller may pass `timeout_secs` to raise.

Adaptive timeouts (codex option D) deferred to PR-2 — brittle regex policy isn't worth shipping in v1.

### Q5 — recursion guard

**[C]** — allow read-only `ato` subcommands (`sessions show`, `dispatches show`, `missions show`, `traces show`, `runtimes health`, `loops show`). Refuse: `dispatch`, `war-rooms`, `missions tick/dispatch/merge`, `replay start`, `loops run`, `methodology run`.

Gemini argued refuse-all simpler; rejected because self-inspection of prior session/dispatch state is genuinely useful and the no-mutation rule keeps it safe.

### Q6 — receipts

**[first-token-only]** — log first-token + arg hash + exit code + duration + stdout/stderr byte counts to `execution_logs.tool_calls_summary`. NOT the raw command (models paste secrets into argv).

Gemini argued full-with-redaction for debugging; deferred to PR-2 as opt-in `--audit-full-command` flag. PR-1.5 default is privacy-first.

### Q7a (codex) — bash argv enforcement

**ADOPTED — non-negotiable.** `bash` executor uses `Command::new(first_token).args(rest)`. **NO `sh -c` invocation.** Without this, `cargo test && rm -rf .` bypasses the first-token check entirely. Caught real security hole.

### Q7b (gemini) — `write_file` parent directory creation

**ADOPTED.** `write_file` auto-creates parent directories via `mkdir -p` semantics. Forcing a separate `mkdir` tool-call is a documented model-failure pattern; small ergonomics win, zero security cost (sandbox_path still applies to the resolved path).

## Receipt shape (per tool)

| Tool | Tracked in `tool_calls_summary` |
|---|---|
| `edit_file` | path, replacement_count, bytes_changed |
| `write_file` | path, bytes_written, was_create_vs_overwrite |
| `list_dir` | path, entry_count |
| `git_status` | result row count (no PII) |
| `git_diff` | ref, path, line_count, bytes |
| `bash` | first_token, arg_hash, exit_code, duration_ms, stdout_bytes, stderr_bytes |

## Open-core split

All 6 tools + sandbox enforcement ship **OSS**. PRO upgrades (queued for `ato-cloud`):

- Team-level deny rules + audit aggregation

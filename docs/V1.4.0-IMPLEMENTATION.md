# v1.4.0 — Production-Grade Agent Authoring: Implementation Guide

This is the executable breakdown of the v1.4.0 plan documented in `ROADMAP.md` and the strategy plan at `~/.claude/plans/peaceful-strolling-kay.md`. Each numbered ticket is a self-contained PR-able unit; **all of them ship together** as a single v1.4.0 release.

**Theme**: turn ATO from "agents-as-static-files" into "agents-as-context-engineered-systems." Inspired by industry consensus on what makes production agents work: dynamic context, specialization, observability. Also where Free / Pro / Team / Enterprise tier gating becomes visible.

**Estimated total**: ~8 weeks of focused work for a single engineer. Internal alpha builds across the way; one big public release.

## Build order

The build order is sequential across waves but parallel within them. **Wave 1 unblocks everything.**

### Wave 1 — Foundations (week 1-2)

#### W1-T1 — SQLite schema additions

**File**: `apps/desktop/src-tauri/src/lib.rs` (`init_database`).

Add to the `execute_batch` block, after the existing `agents` table:

```sql
-- Variables: dynamic prompt resolvers per agent (F1).
CREATE TABLE IF NOT EXISTS agent_variables (
    id          TEXT PRIMARY KEY,
    agent_id    TEXT NOT NULL,
    name        TEXT NOT NULL,
    kind        TEXT NOT NULL,         -- static | env | project-path | file | db-query | mcp-call | computed
    config_json TEXT NOT NULL,         -- JSON specific to the kind
    enabled     INTEGER NOT NULL DEFAULT 1,
    created_at  TEXT NOT NULL,
    updated_at  TEXT NOT NULL,
    UNIQUE (agent_id, name)
);
CREATE INDEX IF NOT EXISTS idx_agent_vars_agent ON agent_variables(agent_id);

-- Pre-call context hooks (F2).
CREATE TABLE IF NOT EXISTS agent_hooks (
    id          TEXT PRIMARY KEY,
    agent_id    TEXT NOT NULL,
    position    INTEGER NOT NULL,      -- ordering within the agent's hook chain
    name        TEXT NOT NULL,
    kind        TEXT NOT NULL,         -- mcp-call | file | db-query | webhook | computed
    config_json TEXT NOT NULL,
    enabled     INTEGER NOT NULL DEFAULT 1,
    created_at  TEXT NOT NULL
);
CREATE INDEX IF NOT EXISTS idx_agent_hooks_agent ON agent_hooks(agent_id, position);

-- Multi-agent groups (F4).
CREATE TABLE IF NOT EXISTS agent_groups (
    id            TEXT PRIMARY KEY,
    slug          TEXT NOT NULL UNIQUE,
    display_name  TEXT NOT NULL,
    description   TEXT,
    runtime       TEXT NOT NULL,        -- runtime the group runs on
    router_config TEXT,                 -- JSON: rules + LLM-classifier config
    file_path     TEXT,                 -- ~/.ato/groups/<slug>/group.json
    created_at    TEXT NOT NULL,
    last_used_at  TEXT
);
CREATE INDEX IF NOT EXISTS idx_groups_runtime ON agent_groups(runtime);

CREATE TABLE IF NOT EXISTS agent_group_members (
    group_id    TEXT NOT NULL,
    agent_id    TEXT NOT NULL,
    role        TEXT NOT NULL,          -- 'router' | 'child'
    position    INTEGER NOT NULL,
    PRIMARY KEY (group_id, agent_id),
    FOREIGN KEY (group_id) REFERENCES agent_groups(id) ON DELETE CASCADE,
    FOREIGN KEY (agent_id) REFERENCES agents(id) ON DELETE CASCADE
);
CREATE INDEX IF NOT EXISTS idx_group_members_agent ON agent_group_members(agent_id);
```

Plus add two columns to the existing `agents` table via separate `ALTER TABLE … ADD COLUMN` statements (SQLite is forgiving; wrap with try-catch in case of repeat runs):

- `role_models_json TEXT` — F5 per-task model selection.
- `memory_policy_json TEXT` — F3 conversation summarizer policy.

In `lib.rs` itself, run those `ALTER TABLE` calls separately after `execute_batch` so existing installs upgrade cleanly.

**Acceptance**: `cargo check` clean; existing local.db opens without error after upgrade.

#### W1-T2 — Cloud migration

**File**: `~/ato-cloud/database/migrations/008_v1_4_0_observability.sql` (new).

```sql
-- Agent traces — every dispatch produces one row (Pro+).
CREATE TABLE IF NOT EXISTS agent_traces (
    id              UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    user_id         UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    agent_slug      VARCHAR(255) NOT NULL,
    runtime         VARCHAR(32) NOT NULL,
    started_at      TIMESTAMPTZ NOT NULL,
    duration_ms     INTEGER NOT NULL,
    ok              BOOLEAN NOT NULL,
    routed_to       VARCHAR(255),                 -- child agent slug if dispatched via group router
    variables       JSONB DEFAULT '{}'::jsonb,    -- resolved variable values
    hooks_fired     JSONB DEFAULT '[]'::jsonb,    -- list of {name, kind, ok, duration_ms}
    prompt_tokens   INTEGER,
    response_tokens INTEGER,
    cost_usd        DECIMAL(10, 6),
    error           TEXT,
    source          VARCHAR(64),                  -- e.g. mcp:run_agent | desktop:run_button
    metadata        JSONB DEFAULT '{}'::jsonb,
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
CREATE INDEX IF NOT EXISTS idx_traces_user ON agent_traces(user_id, started_at DESC);
CREATE INDEX IF NOT EXISTS idx_traces_agent ON agent_traces(user_id, agent_slug, started_at DESC);

-- Evaluator results.
CREATE TABLE IF NOT EXISTS agent_evaluations (
    id            UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    user_id       UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    trace_id      UUID NOT NULL REFERENCES agent_traces(id) ON DELETE CASCADE,
    evaluator_id  TEXT NOT NULL,                  -- user-defined evaluator slug
    kind          VARCHAR(32) NOT NULL,           -- 'heuristic' | 'llm-judge'
    score         DECIMAL(3, 2),                  -- 0.00 - 1.00
    verdict       VARCHAR(32),                    -- 'pass' | 'fail' | 'partial' | 'unknown'
    reason        TEXT,
    cost_usd      DECIMAL(10, 6),
    created_at    TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
CREATE INDEX IF NOT EXISTS idx_evals_trace ON agent_evaluations(trace_id);
CREATE INDEX IF NOT EXISTS idx_evals_user ON agent_evaluations(user_id, created_at DESC);

-- Agent groups (sync mirror of desktop SQLite).
CREATE TABLE IF NOT EXISTS agent_groups (
    id              UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    user_id         UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    team_id         UUID,
    slug            VARCHAR(255) NOT NULL,
    display_name    VARCHAR(255) NOT NULL,
    description     TEXT,
    runtime         VARCHAR(32) NOT NULL,
    router_config   JSONB DEFAULT '{}'::jsonb,
    members         JSONB DEFAULT '[]'::jsonb,    -- [{agent_slug, role, position}]
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    UNIQUE (user_id, slug)
);
CREATE INDEX IF NOT EXISTS idx_groups_user ON agent_groups(user_id);
```

**Acceptance**: `npm run db:migrate` applies cleanly. Tables present in `\dt`.

```ts
export type Tier = 'free' | 'pro' | 'team' | 'enterprise';

export const TIER_ORDER: Record<Tier, number> = {
  free: 0, pro: 1, team: 2, enterprise: 3,
};

export type Feature =
  | 'variables.advanced'   // file/db/mcp/computed resolvers
  | 'context-hooks'
  | 'summarizer.tunable'
  | 'groups.unlimited'     // free is capped at 3 children
  | 'groups.editor'        // editor vs view-only
  | 'role-models'
  | 'cloud-traces'
  | 'evaluators'
  | 'cloud-sync'
  | 'team-workspaces'
  | 'enterprise.evaluator-budgets'
  | 'enterprise.halo'
  | 'enterprise.sso';

export const FEATURE_MIN_TIER: Record<Feature, Tier> = {
  'variables.advanced': 'pro',
  'context-hooks': 'pro',
  'summarizer.tunable': 'pro',
  'groups.unlimited': 'pro',
  'groups.editor': 'pro',
  'role-models': 'pro',
  'cloud-traces': 'pro',
  'evaluators': 'pro',
  'cloud-sync': 'pro',
  'team-workspaces': 'team',
  'enterprise.evaluator-budgets': 'enterprise',
  'enterprise.halo': 'enterprise',
  'enterprise.sso': 'enterprise',
};

export function useTier(): Tier { /* read from useAuthStore */ }
export function useFeatureFlag(f: Feature): boolean { /* tier >= FEATURE_MIN_TIER[f] */ }
```

## OSS / Cloud repo split — what goes where

ATO follows an open-core split between two repos. Critical to keep this clean.

### `~/Agentic-Tool-Optimization/` (OSS, MIT)
Everything that runs **on the user's machine**:

- All desktop Rust + React code (Tauri app)
- All local SQLite schema (variables, hooks, groups, members, all v1.4 tables)
- All resolver / hook / router / dispatch logic (runs locally against the user's runtimes)
- MCP server tools (`list_agents`, `run_agent`, group dispatch)
- Tier-gating UI primitives (`TierGate`, `UpgradePrompt`) — discovery-first; the gates SHOW Pro features to Free users so they can decide to upgrade
- **Pro-tier feature stubs** — db-query, mcp-call, computed resolvers/hooks return clear "Pro implementation server-side" markers from OSS code; the *real* implementation lives cloud-side

The OSS repo is intentionally self-sufficient: all Free-tier functionality works without ato-cloud. Pro features that are *UX-gated* (like multi-agent groups with >3 children) are a soft cap; technically removable by patching OSS, but normal users won't, and the upgrade story is in the conversion funnel.

### `~/ato-cloud/` (closed source, paid)
Everything that requires a server:

- Tier-aware Express services (auth, skills, agent-suggest, agent-traces, etc.)
- PostgreSQL schema + migrations (cloud-side mirror of agent records, plus traces/evaluations)
- `requireTier` middleware enforcing Pro/Team/Enterprise gating server-side
- Pro implementations of the OSS-stubbed resolvers/hooks (`POST /resolve-variable`, `POST /run-hook`)
- LLM-as-judge evaluator service (Wave 4)
- HALO trace-optimization service (Enterprise, v1.6)
- SSO / audit retention / SOC2 plumbing (Enterprise)
- Hosted templates library, backup vault, hosted PTY (Team tier)

### The honest tier gate

For local-only features (variables, hooks, summarizer, models, groups), the OSS gate is **discovery UX**, not capability enforcement. A determined user could fork and remove the lock — that's fine. Open-core accepts this.

For cloud-dependent features (cross-device sync, hosted suggest, evaluator runs, traces retention, team workspaces, HALO, SSO), the gate is **server-enforced** via `requireTier`. No bypass.

This split is reflected in every Pro stub in the OSS codebase — when a user with no Pro tier hits a stubbed resolver, the stub returns a clear `"pro-feature-not-yet-implemented"` marker. Wave 2.2+ replaces these stubs with HTTP calls to the cloud:

```
Free path:        OSS resolver → returns marker
Pro path (Wave 2.2): OSS resolver → POST https://api.agentictool.ai/resolve-variable → cloud runs the actual db-query/mcp-call/computed
```

The OSS code only ever knows how to *call* the cloud endpoint; the real implementation never ships in MIT.

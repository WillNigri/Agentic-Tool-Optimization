## Known v2.15 follow-up items (do NOT mark as regressions)

- **CLI `--encrypted` append**: returns clean refusal error — Rust libsodium port is a Wave 4 follow-up.
- **PRO Runner sidecar binary**: not built yet — the OSS shim throws `ProRunnerNotInstalledError`. The download endpoint + signed binary land in a follow-up.
- **Loop Composer runner**: 7 of 9 node kinds (`diagnose`/`apply`/`review`/`war_room`/`score`/`input`/`output`) write `status="skipped"`. v2.14.1 cluster; tracked.
- **Forward secrecy on E2E**: out of scope. Team Key rotation on member removal + privkey rotation only.
- **Standalone web client**: deferred to v2.16+ (read-only) → v2.17 (tethered to desktop) → v2.18+ (standalone with browser-stored key, only if v2.17 doesn't carry it).
- **CSO M (CLI auth.json file permissions)**: file is `0644` by default; should be `0600` or moved to OS Keychain. Tracked as a hardening follow-up.
- **CSO M (anon telemetry secondary in-route rate limit)**: gateway-only rate limit today; secondary in-route limiter is a hardening follow-up.
- **CSO H4 server-side serializable txn around mode-flip + append**: client-side pre-flight closes the common case; cloud-side txn deferred.
- **Library spec drift**: v2.15 spec table mentioned `@stablelib/*`; actual implementation is `libsodium-wrappers`. Spec doc to update.

# ATO v2.3.32 — v2.3.51: The Phase 6 Cluster

**Released**: May 12, 2026
**Scope**: Cross-runtime conversations, eval-score ratchets, runtime health, API streaming, MCP expansion. 21 incremental releases shipped over the cluster; v2.3.51 is the rollup release tag.

---

## Headline

**ATO is now a credible workflow ops platform for multi-runtime AI agents.** Phase 6 closes a real loop: agents on different runtimes (Claude / Codex / Gemini / MiniMax / Grok / DeepSeek / Qwen / OpenRouter / OpenClaw / Hermes — local or SSH-remote) can hold a single conversation, hand work to each other by `@`-mention, and the quality of each runtime's contribution gets locked as a CI gate.

If you've been wanting the answer to "would this agent be cheaper / better on a different runtime?" — that's the whole arc.

## What's new

### Cross-runtime agent conversations (Phase 6 Slices A → B)

```bash
ato sessions new --runtime claude --title "ssh-adapter design review"
ato dispatch claude "..." --session <id>          # claude runs, history captured
ato dispatch minimax "..." --session <id>         # minimax replies, sees claude's turn
ato dispatch claude "..." --session <id> --tag-bridge   # auto-bridge any @mentions
```

- **Sessions** — sticky multi-turn conversations persisted to local SQLite. Native `--resume` when the runtime supports it (claude), history-replay everywhere else.
- **Bridge** — when an assistant turn contains `@runtime-name`, the next dispatch into that session auto-routes to the named runtime. Loops until `[CONSENSUS]` (or the structured `<consensus/>` tag) on a line by itself, no further mention, or `--max-rounds` (default 3).
- **Spinning detector** — heuristic: if the same runtime produces 3 consecutive assistant turns inside one bridge run, the bridge stops dispatching and posts an `approval_request` to the activity feed for human review. Not a semantic "is the conversation productive?" detector — just a structural backstop against monologue loops.
- **GUI surface** — "Sessions" tab under Runs lists every session with runtime badges, click into a chat-style transcript view, type into the Continue input or hit the Bridge button. New Session modal opens a fresh conversation. Streaming API providers render chunks live in the transcript as they arrive.

### Eval-score ratchet — CI gate for agent quality (Phase 6.x-K)

Inspired by Garry Tan's [AI Agent Complexity Ratchet](https://garrytan.com/) post: once you lock a quality floor, no future dispatch can drop below it without the CI step failing loudly.

```bash
ato ratchet lock --target runtime:claude --days 30 --threshold 0.05
ato ratchet check                                  # exit 1 on any breach
ato ratchet check --post-on-fail                   # also write to activity feed
ato ratchet status                                 # informational, exit 0
```

- Per-target floor: `runtime:<name>` / `agent:<slug>` / `global`.
- Computed from local `execution_logs.status` (`success_rate` metric for v1; schema discriminator already in place for `eval_score` when local evals land).
- Pre-flight warning in `ato dispatch` when the rolling rate is within 1pp of the floor — surfaces "this dispatch failing may breach the lock" before you commit credits.
- GUI panel in Insights → Ratchet visualizes locked floors, current rates, and breach history. Lock / unlock from the GUI shells out to the same CLI.
- Breach events publish to `events_log` so ops recipes can react (`on ratchet_breach` triggers).

### SSH-backed remote runtime adapter (Phase 6.x-J)

```bash
ato runtimes add-remote \
    --name claude-server \
    --host you@your-server \
    --runtime claude \
    --binary-path claude
ato dispatch claude-server "review this..."        # routes over SSH
```

One-way runtime invocation from a laptop to a server (or any reachable SSH host). The remote ATO daemon isn't required — the laptop owns sessions, logs, and bridging; the remote just executes the binary. Persistence shape is identical to local dispatch so audit / sessions / regressions / cost panels see the remote run as just another row. Phase 7+ bi-directional daemon mesh — server-initiated calls back to the laptop, mDNS / cloud-relay discovery, full wire protocol — is roadmap-noted but not built. The SSH adapter handles the most common laptop → server case; the reverse direction is a deliberate next-scoping decision once a real user asks for it.

Direct answer to [@iamknownasfesal](https://x.com/iamknownasfesal)'s X question about bridging laptop ↔ server Claude installs.

### Runtime-binary health check (Phase 6.x-I / I.2)

```bash
ato runtimes health
```

Detects when a runtime CLI is broken before you spawn it and macOS Gatekeeper pops a malware dialog:

- **Revoked Developer ID cert** (the CSSMERR_TP_CERT_REVOKED case that bricked codex on the maintainer's machine 2026-05-11 — the trigger for shipping this).
- **Quarantined** (`com.apple.quarantine` xattr).
- **JS-shim sidecar walking**: npm-installed runtimes like `codex` are `#!/usr/bin/env node` scripts that can't themselves be signed. The health check now follows the shim into `node_modules/<vendor>/<pkg>-darwin-<arch>/` (including nested layouts like codex's `vendor/<triple>/<runtime>/<runtime>`) and verifies the actual Mach-O sidecar.

Desktop banner pinned to Home with a "Run fix" button that executes the canned fix command (`npm install -g <pkg>@latest` or `xattr -d com.apple.quarantine <path>`) through an IPC allowlist. No `sh -c` of untrusted strings.

### API provider streaming (Phase 6.x-F)

```bash
ato dispatch minimax "long response..." --stream         # raw chunks to stdout
ato dispatch minimax "..." --stream-jsonl                # JSONL events for wrappers
```

Non-CLI providers (MiniMax / Grok / DeepSeek / Qwen / OpenRouter) had been buffering 7–15s of output. Streaming closes that gap with SSE parsing; tokens-in/out captured from the final usage chunk so the audit row matches the buffered path. GUI Sessions transcript renders chunks live for these providers.

### MCP server: 8 → 52 tools

- Added: sessions / bridge / posts / activity-feed / runtime health / remote runtimes / events / ratchet (lock / check / list)
- Every new tool wraps the canonical CLI implementation — single algorithm, two surfaces (CLI + MCP), schema migrations tested in one place.
- v2.3.51 ships `npm run qa:stdio` — a Node-based smoke test that drives the MCP server over real stdio JSON-RPC, lists tools, invokes a couple, asserts response shapes. Caught a real ATO_CLI_PATH gap in the process.

### `ato-review` Claude Code skill

`.claude/skills/ato-review/SKILL.md` ships in the repo. When a Claude Code session opens this repo, the skill instructs Claude to dispatch every non-trivial diff to a reviewer runtime via `ato dispatch <reviewer> --session review/<branch>` before committing — parse findings, apply or defer with justification, record the audit trail in the commit body.

Portable: copy `.claude/skills/ato-review/` into any repo, set `$ATO_REVIEWER`, done. Codifies what Tan calls the complexity ratchet — "build passes ≠ reviewed" — as a contract Claude follows automatically.

## What this isn't

- **Not a daemon mesh** — the SSH adapter is one-way (laptop initiates). Phase 7+ bi-directional discovery (mDNS / cloud relay / wire protocol for server-initiated calls back) is on the roadmap as blue-sky until a real user asks for it.
- **Not a production agent SDK** — ATO is still positioned for the developer workflow / ops loop. Langfuse, Helicone, and the existing observability vendors cover the production sidecar story. ATO sits next to them, not in their lane.
- **Not deterministic on Slice B `[CONSENSUS]` termination** — relies on the model phrasing agreement as the bracketed/tagged form the bridge cue asks for. Tested across several internal cross-runtime sessions on this build; QA.md §5 flags it as needing a mock-LLM fixture for true regression protection, targeted for v2.4.

## Try it

```bash
# Install
npm install -g @ato/desktop   # or download from releases
ato setup-path                # makes `ato` available shell-wide

# Open a cross-runtime conversation
SID=$(ato sessions new --runtime claude --title "demo" | jq -r .id)
ato dispatch claude "Review this design: <paste>. @minimax also weigh in." --session $SID --tag-bridge

# Lock a quality floor + add it to CI
ato ratchet lock --target runtime:claude --days 30
ato ratchet check                                   # exit non-zero on breach

# Hook up a remote runtime
ato runtimes add-remote --name claude-server --host you@server --runtime claude

# Check your runtimes are signed + reachable
ato runtimes health
```
